# Modification of Instances
Instances can be found in the folder [`instances/vrp`](instances/vrp).
- **Vehicle capacities**  
  - `diesel`: uses the original capacity from the instance  
  - `clean`: uses **90% of the original capacity**

- **Fleet composition**  
  - Total fleet size: **20 vehicles**  
  - Evenly split between types:
    - 10 `diesel` vehicles  
    - 10 `clean` vehicles  

- **Other parameters**  
  - All remaining instance data (customer locations, demands, etc.) remain unchanged from the original instances

# Weighted-sum Green CVRP workflow

This implementation is provided in the file [`weighted_sum.py`](weighted_sum.py). It solves a bi-objective Green CVRP with two vehicle types (`diesel` and `clean`) using a weighted-sum approach on top of PyVRP.

The code follows this workflow:

1. Define global parameters for fuel consumption, CO2 emissions, fleet composition, runtime, and weighted-sum coefficients.
2. Read a VRP instance from file.
3. Build a PyVRP model with two vehicle types.
4. For each weight pair `(w_dist, w_co2)`:
   - convert the bi-objective problem into a single surrogate routing cost
   - solve the resulting VRP with PyVRP
   - evaluate the true load-dependent CO2 emissions of the obtained solution
   - store distance, CO2, and solution information
5. Identify nondominated solutions to approximate the Pareto front.
6. Plot the objective-space trade-off.
7. Select one Pareto solution and visualize its routes, colored by vehicle type.

# Approach 1 of Green CVRP with NSGA-II workflow
This implementation is provided in the file [`clean_nsga2.py`](clean_nsga2.py). It solves a bi-objective Green CVRP with two vehicle types (`diesel` and `clean`) using NSGA-II algorithm.

## Chromosome Representation
Each NSGA-II individual is represented as a **permutation of all customers**.
This means that the chromosome only stores the order in which customers should be considered. Each customer appears exactly once.

The code follows this workflow:

1. Define global vehicle and emission parameters:
   - vehicle capacities,
   - number of diesel and clean vehicles,
   - CO₂ multipliers,
   - fuel consumption parameters.
2. Read the VRP instance and extract:
   - distance matrix,
   - customer demands,
   - depot index.
3. Create a `GreenInstance` object containing:
   - problem data,
   - fleet information,
   - emission model parameters.
4. Generate an initial solution using PyVRP:
   - build a PyVRP model with two vehicle types,
   - solve the instance for a short runtime,
   - extract the best PyVRP solution for distance,
   - convert its routes into a customer permutation.
5. Represent each NSGA-II individual as a chromosome:
   - a permutation of all customers,
   - where each customer appears exactly once.
6. Create the initial NSGA-II population:
   - first individual is based on the PyVRP solution,
   - remaining individuals are generated by slightly perturbing this permutation.
7. Decode each chromosome into routes:
   - split the customer permutation into feasible route segments,
   - assign each route to a feasible vehicle type,
   - prefer the vehicle type with lower CO₂ when possible.
8. Evaluate each solution using two objectives:
   - total traveled distance,
   - total load-dependent CO₂ emissions.
9. Apply NSGA-II operators:
   - order crossover,
   - swap mutation,
   - nondominated sorting,
   - crowding-distance-based selection.
10. Return the final approximation of the Pareto front:
    - objective values `[distance, CO₂]`,
    - decoded routes for selected solutions,
    - vehicle type used by each route.
